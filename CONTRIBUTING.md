# Contributing Guide

Welcome! By working out of this repository, you will simulate how developers contribute to Open Source and Enterprise projects.

## Prerequisites
- A GitHub Account
- Git installed on your local machine

## Ground Rules
1. **Never commit directly to `main`.** Always create a new branch for your work.
2. **Write descriptive commit messages.** Instead of `"fixed stuff"`, write `"fix: typo in contributors file"`.
3. **Open a Pull Request (PR).** Once your changes are pushed to your remote branch, open a PR against this repository's `main` branch to get your work merged.

## Initial Setup
Before starting the modules, you need your own copy of this repository.

1. **Fork the Repository:** Click the "Fork" button at the top right of this page on GitHub to create a copy under your account.
2. **Clone your Fork:**
   ```bash
   git clone https://github.com/<YOUR_USERNAME>/git-repo-2026.git
   cd git-repo-2026
   ```
3. **Add the Original Repo as 'Upstream' (Optional but recommended):**
   ```bash
   git remote add upstream https://github.com/<ORIGINAL_OWNER>/git-repo-2026.git
   ```

Now, head back to `README.md` and start **Module 1**!
